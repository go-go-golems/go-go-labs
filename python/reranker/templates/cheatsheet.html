Reranker Cheatsheet
YAML Format

Your YAML must contain the following structure:

query: "Your query text here"
documents:
  - "First document text"
  - "Second document text"
  - "Third document text"
top_k: 5  # Optional, limits results to top K documents
About the Model

This application uses the BAAI/bge-reranker-large model to rerank documents based on their relevance to a query. The reranker calculates relevance scores for each (query, document) pair and sorts them by score.

How Reranking Works

Document reranking is a core component of modern retrieval systems:

  1. You provide a query and a set of candidate documents
  2. The reranker evaluates the relevance of each document to the query
  3. Documents are sorted by relevance score (higher is better)

Unlike retrieval models, rerankers work with a limited set of documents and focus on precision rather than recall.

Tips for Better Results
  • Be specific with your queries
  • Provide enough context in your documents
  • Experiment with different query phrasings
  • Documents should be self-contained semantic units
Technical Details
  • The model uses a cross-encoder architecture that jointly processes query-document pairs
  • Maximum input length is 512 tokens (longer inputs will be truncated)
  • Performance may vary based on document length and content similarity
